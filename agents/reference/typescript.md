# TypeScript Style Guide

TypeScript coding standards for workflow-agents.

## Naming Conventions

- **Variables**: `camelCase`
- **Constants**: `UPPER_SNAKE_CASE`
- **Functions**: `camelCase`
- **Classes**: `PascalCase`
- **Interfaces**: `PascalCase` (no `I` prefix)
- **Types**: `PascalCase`
- **Enums**: `PascalCase`

## Type Annotations

Always use explicit types for function signatures:

```typescript
// Good
function processData(input: string): ProcessedResult {
  // ...
}

// Avoid
function processData(input) {
  // ...
}
```

## Interfaces vs Types

Prefer interfaces for object shapes:

```typescript
interface User {
  id: string;
  name: string;
  email: string;
}
```

Use types for unions, intersections, utilities:

```typescript
type Status = 'pending' | 'active' | 'completed';
type PartialUser = Partial<User>;
```

## Best Practices

- Enable strict mode in tsconfig.json
- Avoid `any` - use `unknown` if type truly unknown
- Use readonly for immutable properties
- Prefer const assertions for literal values
- Document complex types with JSDoc

## File Organization

```
src/
├── types/          # Type definitions
├── interfaces/     # Shared interfaces
├── models/         # Data models
├── services/       # Business logic
└── utils/          # Utility functions
```
