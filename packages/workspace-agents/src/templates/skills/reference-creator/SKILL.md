# Reference Creator

Create focused, topic-specific reference documentation that provides deep guidance for AI agents.

## Overview

This skill provides tools to create well-structured reference documents following the framework's documentation patterns.

**Core capabilities**:
1. **create** - Generate new reference doc from template
2. **validate** - Check reference completeness

## Commands

### Create a New Reference Doc

Generate a reference document with interactive prompts:

```bash
cd agents/skills/reference-creator
npm run create
```

**Prompts**:
- Topic name (e.g., testing, database, api-design)
- Brief description
- Reference type (language, framework, workflow, general)
- Key sections to include

**Output**: Creates reference file at `agents/reference/<topic>.md`

### Validate a Reference Doc

Check reference completeness and structure:

```bash
cd agents/skills/reference-creator
npm run validate path/to/reference.md
```

**Validation checks**:
- Document structure follows template
- Headers are properly nested
- Code examples have language specified
- Links are valid
- Length is under 500 lines

## When to Create a Reference Doc

Create a new reference doc when:
- A topic requires deep documentation (language patterns, framework usage)
- Information doesn't fit in AGENTS.md (too detailed)
- Multiple agents/personas need the same context
- Best practices need to be documented
- Complex workflows need step-by-step guidance

**Don't create reference docs for**:
- One-off tasks (use plans instead)
- Executable actions (use skills instead)
- Agent personalities (use personas instead)
- Active work tracking (use tasks.md)

## Reference Types

### Language Reference

**When to use**: Documenting language-specific patterns, idioms, and standards

**Example topics**:
- `typescript.md` - TypeScript patterns and configuration
- `python.md` - Python idioms and project structure
- `rust.md` - Rust ownership and error handling

**Template sections**:
- Language Configuration
- Code Style
- Common Patterns
- Error Handling
- Testing Patterns
- Best Practices

### Framework Reference

**When to use**: Documenting framework-specific usage and patterns

**Example topics**:
- `react.md` - React component patterns
- `express.md` - Express middleware and routing
- `fastapi.md` - FastAPI endpoint patterns

**Template sections**:
- Setup and Configuration
- Core Concepts
- Common Patterns
- Testing
- Performance Tips

### Workflow Reference

**When to use**: Documenting multi-step processes and workflows

**Example topics**:
- `plan_act.md` - Plan → Act → Test workflow
- `code-review.md` - Code review process
- `deployment.md` - Deployment workflow

**Template sections**:
- Overview
- Step-by-Step Process
- Decision Points
- Common Issues
- Examples

### General Reference

**When to use**: Documenting cross-cutting concerns

**Example topics**:
- `documentation-style.md` - Documentation standards
- `agent-patterns.md` - Agent development patterns
- `security.md` - Security guidelines

**Template sections**:
- Overview
- Guidelines
- Examples
- Best Practices

## Reference Template

```markdown
# Topic Name

Brief overview of what this reference covers and when to consult it.

## Overview

Expand on the purpose and scope. What does this reference help with?

## [Main Section 1]

### Subsection

Content with examples:

```language
// Code example
```

### Subsection

More content.

## [Main Section 2]

Continue with topic coverage.

## Best Practices

- **Practice 1**: Explanation
- **Practice 2**: Explanation
- **Practice 3**: Explanation

## Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| Issue 1 | Why it happens | How to fix |

## Related Resources

- [Related Skill](../skills/skill-name/) - Relationship
- [Related Persona](../personas/persona-name.md) - Relationship

---

*Generated by workspace-agents on YYYY-MM-DD*
```

## Naming Convention

**Use kebab-case topic names**:

- `typescript.md`
- `api-design.md`
- `code-review.md`
- `error-handling.md`

## Reference vs Other Resources

**Reference docs** (`agents/reference/`):
- Deep documentation on topics
- Read for context and guidance
- Relatively static
- Example: "How to structure TypeScript projects"

**Skills** (`agents/skills/`):
- Executable, deterministic actions
- DO things - create files, validate, package
- Example: "Create a new reference doc"

**Personas** (`agents/personas/`):
- Agent personalities and expertise
- GUIDE how and when to use resources
- Example: "Documentation agent knows documentation standards"

**Plans** (`agents/plans/`):
- Implementation planning
- Track active work
- Evolve during implementation
- Example: "Plan for adding user auth"

## Validation Checklist

Before finalizing a reference doc:

- [ ] Topic is specific and focused
- [ ] Overview explains when to consult this doc
- [ ] Headers are properly nested (H1 → H2 → H3)
- [ ] Code examples specify language
- [ ] Examples are concrete and actionable
- [ ] Best practices are included
- [ ] Common issues are documented
- [ ] Related resources are linked
- [ ] Document is under 500 lines
- [ ] No duplicate content from AGENTS.md
